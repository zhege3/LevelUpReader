# Changelog

## v1.1.0 — 2026-06-18

### 多角色识别重写 (ScriptParser / CharacterAnalyzer)

**旧逻辑问题：** 搜索窗口固定 40/20 字符，相邻对白串话严重；贪婪正则会吞掉"说/道"作为名字的一部分；After 窗口优先级不当。

**新逻辑：**

| 层级 | 改进 |
|---|---|
| **搜索窗口** | 段落边界 (`\n\n`) + 相邻对白间距约束，不再固定窗口 |
| **Tier 1** | `动词+名字` 在 after 窗口，最近优先匹配 |
| **Tier 2** | `名字+动词` 同时搜索 before + after 窗口，距离优先 |
| **Tier 2.5** | **新增** `名字…：` 模式，覆盖无说话动词场景（如"张三笑了一声："） |
| **Tier 3** | before 窗口 (15 字符) 优先，after 窗口 (8 字符) 仅作回退 |
| **Tier 4** | 继承检测：before 窗口内出现其他角色则阻断继承 |
| **CharacterAnalyzer** | `{2,4}` → `{2,4}?` 修复贪婪捕获吞词 |

### 朗诵间隔优化 (TTSWorker)

- 新增 `_gen_async` 后台预生成：播放当前句时异步生成下一句 MP3
- 除首句外，句间间隔从 ~2 秒降至接近 0

### 精简模式支持窗口缩放

- 鼠标移至窗口边缘/四角 8px 范围显示双向箭头
- 拖拽边缘可调整窗口大小（最小 200×100）
- 中间区域保持拖动移动

### 测试

- 30 项单元测试覆盖：相邻串话修复、段落隔离、距离优先、续接继承、新角色打断、冒号模式识别等场景，全部通过
